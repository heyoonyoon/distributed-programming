# UC13. 보험 가입을 심사한다

**액터:** 보험사 직원

---

## 선행 조건

- 보험사 직원이 시스템에 로그인하여 심사 권한을 인증받은 상태여야 한다.
- [UC02. 보험 가입을 요청한다]가 완료되어 '심사 대기' 상태의 신청 정보가 존재해야 한다.

## 후행 조건

심사 결과(승인/조건부 승인/반려), 처리 일시, 담당자 정보가 저장되며, 가입자에게 결과 통보가 완료된다.

---

## 베이직 플로우

1. 보험사 직원이 메인 화면에서 '심사 대기 목록' 메뉴를 클릭한다.
2. 시스템은 현재 심사 대기 중인 목록(접수일시, 가입자명, 보험 종류, 상품명, 보험료 등)을 화면에 출력한다.
3. 보험사 직원이 목록에서 특정 심사 건의 '심사하기' 버튼을 누른다. (E3)
4. 시스템은 해당 건의 상세 정보(이름, 생년월일, 주민등록번호), 보험 종류별 추가 정보(차량 정보 또는 의료 고지 항목), 신청 보장 내용을 화면에 표시한다.
5. [확장 유스케이스: UC15. 자동차 사고 이력을 조회한다] 를 수행한다. ※ 자동차보험 건에 한하여 조건부 실행 (E1)
6. 시스템은 조회된 사고 건수, 유형, 지급액 합계, 면허 상태 정보를 심사 화면의 '참조 정보' 영역에 출력한다.
7. 보험사 직원이 조회 정보와 가이드라인을 검토한 후, 심사 결과 항목에서 '승인' 버튼을 선택하고 '심사 확정' 버튼을 클릭한다. (A1, A2, E2)
8. 시스템은 심사 결과, 처리 일시, 담당자 정보를 저장한다.
9. 시스템은 가입자에게 안내 이메일/문자를 발송하고, 화면에 "심사가 완료되었습니다." 메시지를 출력한다.

---

## 대안 흐름

### A1. 조건부 승인 — 보험료 할증이 필요한 경우 (7단계 분기)

- **A1-1.** 보험사 직원이 심사 결과로 '조건부 승인'을 선택한다.
- **A1-2.** 보험사 직원이 할증율(예: +20%)과 할증 사유를 입력하고 '보험료 재계산' 버튼을 누른다.
- **A1-3.** 시스템은 할증된 최종 보험료를 화면에 다시 계산하여 보여준다.
- **A1-4.** 보험사 직원이 산출된 금액을 확인하고 '최종 확정' 버튼을 클릭한다.
- **A1-5.** 시스템은 조건부 승인 결과와 변경된 보험료를 저장하고 가입자에게 안내를 발송한다.

### A2. 타 부서 협의 후 심사 결정 (7단계 분기)

- **A2-1.** 보험사 직원이 심사 화면에서 '부서 협의 요청' 버튼을 누른다.
- **A2-2.** 보험사 직원이 협의 사유를 입력하고 대상 부서(예: 의료심사팀)를 선택한 뒤 '요청 전송'을 클릭한다.
- **A2-3.** 시스템은 해당 건의 상태를 '협의 중'으로 변경하고 협의 부서 담당자에게 알림을 보낸다.
- **A2-4.** 협의 부서 담당자가 검토 의견을 입력하면, 시스템은 원 담당자에게 알림을 발송한다.
- **A2-5.** 보험사 직원이 협의 의견을 확인한 후 최종 결과를 결정하여 8단계로 돌아간다.

## 예외 흐름

### E1. 외부 기관(금융감독원) 정보 연동 실패 (5단계에서 발생)

- **E1-1.** 외부 시스템 응답 지연 또는 오류 발생 시, 시스템은 "정보 조회 실패" 안내창을 띄운다.
- **E1-2.** 보험사 직원이 안내창에서 '재시도' 또는 '직접 입력 전환' 버튼을 클릭한다.

### E2. 심사 확정 후 저장 실패 (7단계에서 발생)

- **E2-1.** 정보 저장 중 오류 발생 시, 시스템은 "저장 실패" 메시지를 출력한다.
- **E2-2.** 시스템은 직원이 입력한 심사 의견 및 결과 값이 사라지지 않도록 입력 내용을 유지한다.
- **E2-3.** 보험사 직원이 다시 '심사 확정' 버튼을 눌러 재시도한다.

### E3. 동일 심사 건에 중복 접근 시도 (3단계에서 발생)

- **E3-1.** 보험사 직원 B가 이미 처리 중인 건의 '심사하기' 버튼을 누른다.
- **E3-2.** 시스템은 "현재 담당자 [ID]가 처리 중인 건입니다."라는 경고 메시지를 띄우고 상세 화면 진입을 차단한다.
