# 발표 자료 작업 규칙 (반드시 지킬 것)

대상: `보험발표-figma.pptx` (대학 기말 발표 덱). 아래는 예외 없는 하드 규칙이다.

## 1. 한 슬라이드 = 한 메시지 — 정보량 최소화 (최우선)
- 슬라이드 하나에 개념 하나. 박스·텍스트·줄 수를 최소로.
- 코드 슬라이드는 스니펫 1개 + 한 줄 설명. 글머리표 나열 금지.
- "더 넣고 싶다" 싶으면 페이지를 쪼갠다. 채우지 말고 비운다.

## 2. 기존 파일 in-place 수정 — 전체 재생성 금지
- 사용자가 PowerPoint에서 직접 편집한다. **`build_figma.py`로 전체 덮어쓰기 절대 금지** (편집이 날아감).
- 고칠 슬라이드만 골라 제자리에서 다시 그린다(`edit_*.py`: 제목 텍스트로 슬라이드를 찾아 그 슬라이드만 clear 후 재작도).
- 작업 전 `git diff --quiet 보험발표-figma.pptx`로 사용자 편집 여부 확인. 파일이 열려 있으면 닫아 달라고 먼저 요청.

## 3. 텍스트
- 영어 소제목(eyebrow) 금지. 제목·본문 폰트 크게(멀리서 보임).
- 영어·전문용어 대신 쉬운 한국어. 꼭 영어를 쓸 코드/식별자만 예외.
- 줄바꿈으로 잘리지 않게(박스 폭·폰트 점검).

## 4. 디자인
- 절제된 인디고 단일 액센트 + 뉴트럴. 알록달록 금지.
- 설계 다이어그램 = 베이지(EA풍), 코드 다이어그램 = 인디고. 둘을 시각적으로 구분.
- 그림자 없음, flat. 박스 안 잘리게.

## 5. 검수
- 수정 후 항상 `qlmanage`로 해당 슬라이드 PNG 렌더해 눈으로 확인하고 커밋.
