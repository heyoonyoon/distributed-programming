# AI 자율 추가 금지 규칙

## 이 규칙의 목적

AI가 "더 좋게 만들려고" 설계에 없는 것을 추가하는 행위를 막는다.
AI가 제안을 하더라도, 사용자의 명시적 승인 없이는 구현하지 않는다.

## 절대 금지 목록

### 기술 추가 금지
- `import org.springframework.*` → 금지
- `import javax.persistence.*` → 금지
- `import java.sql.*` → 금지
- `import com.fasterxml.jackson.*` → 금지
- `Dockerfile`, `docker-compose.yml` 생성 → 금지
- `build.gradle`에 새 dependency 추가 → 금지 (사용자 승인 없이)

### 구조 추가 금지
- 설계서에 없는 클래스 생성 → 금지
- 설계서에 없는 메서드 추가 → 금지
- 설계서에 없는 필드 추가 → 금지
- 인터페이스/추상클래스 임의 추가 → 금지
- 유틸리티 클래스 임의 생성 → 금지

### 패턴 임의 적용 금지
- Repository 패턴 → 금지 (DB 없음)
- Service 레이어 임의 분리 → 금지 (다이어그램 따름)
- DTO 임의 생성 → 금지 (다이어그램에 없으면)
- Exception 클래스 임의 생성 → 금지

## Claude가 해야 할 것

Claude가 뭔가 추가하고 싶을 때는:
1. 구현을 **멈춘다**
2. 사용자에게 이렇게 묻는다:

```
[질문] 설계서에 없는 요소가 필요해 보입니다.
- 추가하려는 것: [구체적 내용]
- 이유: [왜 필요한지]
- 진행할까요? (Y/N)
```

3. 사용자가 N 또는 무응답이면 **설계서대로만** 진행한다.

## 예외 (허용되는 것)

- `main()` 메서드의 메뉴 텍스트 문구 (흐름이 같다면)
- `System.out.println` 출력 메시지 문구
- `Scanner` 입력 안내 문구
- 주석(comment) 추가
